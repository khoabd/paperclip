## Architecture Decision Records

- [[ADR-0002-Pure-LangGraph]] — *Historical (Python era).* Superseded for TS by ADR-0003. (2026-04-24)
- [[ADR-0003-Strategic-Loop-Runtime]] — Custom TS state machine + Drizzle persistence over LangGraph-JS for Strategic Loop / Greenfield / Intake flows. (2026-04-29)
- [[ADR-0004-Magika-Sidecar]] — Magika file triage runs as Python sidecar plugin (`plugin-magika`) with batch loopback HTTP, reusing paperclip plugin contract. (2026-04-29)
- [[ADR-0005-DBSCAN-Library]] — `density-clustering` npm with custom cosine wrapper for feedback / rejection / intake clustering. eps=0.25, minPoints=3. (2026-04-29)
- [[ADR-0006-Workspace-Equals-Company]] — Workspace 1:1 maps to existing `companies` table; no new workspace table. UI label "Workspace", code field `companyId`. Platform layer is its own slice. (2026-04-29)
- [[ADR-0007-Brain-Storage]] — Per-workspace Brain stored as `documents` row with `key='brain'`; reuses revisions + embeddings. Sub-artifacts use `brain/personas`, `brain/principles`, etc. (2026-04-29)
- [[ADR-0008-Drag-In-Detection]] — V1 self-report only ("was this a drag-in?" toggle on approval responses). V2 hybrid heuristic deferred until 3 months of self-report data. (2026-04-29)
- [[ADR-0009-Approvals-Architecture]] — Extend existing `approvals` table (do NOT create `approval_items`). Adds proposal_pattern/confidence/risk_score/priority/timeout/delegation columns; pattern-specific shapes carried in `payload` JSONB with Zod discriminated union. (2026-04-29)
- [[ADR-0010-MCP-Client-Framework]] — Inbound MCP framework lives in new `packages/mcp-client/` (separate from the existing outbound `packages/mcp-server/`). Adapters (`gitlab-mcp`, `opensearch-mcp`, `research-mcp`) wrap typed Zod inputs around the generic client. Built-in retry, circuit breaker, request redaction, audit recorder. (2026-04-29)
