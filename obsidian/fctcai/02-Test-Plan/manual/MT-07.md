# MT-07 — Design conflict UX

## Mục tiêu
Verify conflict warning hiện trước khi user có thể approve design doc B (cùng component path với A).

## Steps
1. Tạo design_doc A cho `auth/oauth.ts` status=in_dev
2. Mở form tạo design_doc B cùng path
3. Verify warning "Conflict with design doc A" hiện ra
4. Try submit B → block với inline error
5. Resolve A (set status=merged) → submit B succeed

## Expected
- Warning hiện realtime khi user nhập path
- Submit blocked nếu conflict unresolved
- Path autocomplete suggests existing design docs
